注入类漏洞

  • CWE-89 SQL 注入

这些漏洞的核心问题都在于不安全的输入处理，攻击者通过构造特定的输入来改变程序的正常执行流，导致未授权的操作或数据泄露

上述漏洞类型都符合 source-sink，可以使用污点分析获取代码片段

CWE-79 XXS

XSS 的 sink 点判定要麻烦一些，上面收集的不全，只能手搓了
可以通过判断 java 的 io api 判断 ma.getMethod() instanceof WritingMethod

但是只判断某个方法是否属于 WritingMethod（即一个写操作）。然而，仅依赖这一条件会产生大量误报，因为即使是 WritingMethod，也不一定意味着存在 XSS 问题，因此加入条件XssVulnerableWriterSourceToWritingMethodFlow::flowToExpr(ma.getQualifier())。
这个条件是核心，因为它确保数据流分析系统已经确定某个“污染源”的数据从源头（如 getWriter）流到 WritingMethod，而不仅仅是一个普通的写操作。只有当有明确的数据流路径时，才会触发警报。
